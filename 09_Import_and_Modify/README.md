# Prompt-Based CAD Modification Workflow

This folder demonstrates one of the most powerful capabilities of an automated AI agentic workflow: **The ability to import existing, standard CAD files (.step) and modify them computationally based on natural language prompts.**

Unlike STL mesh files (which are extremely difficult to modify parametrically), `.step` files retain their precise boundary representations. This allows frameworks like `build123d` to load them, select specific geometric features (faces, edges, vertices), and perform complex boolean operations (drilling, slicing, filleting) on them as if they were drawn from scratch.

This directory contains two distinct workflows to prove this capability:

---

## Workflow 1: The "Simulated" Download & Edit
Because we didn't have a reliable internet download at first, we simulated the process of finding an online file and modifying it.

1. **`generate_blank_bracket.py`**
   - **What it does:** Generates a very generic, blank L-bracket from scratch and exports it as `blank_bracket.step`.
   - **Purpose:** Acts as our "downloaded" file (e.g., an unedited structural part downloaded from McMaster-Carr or GrabCAD).
   
2. **`modify_imported_step.py`**
   - **What it does:** Autonomously imports `blank_bracket.step`, computationally analyzes the geometry to find the flat top face of the base, and drills a precise 2x2 grid of 6mm mounting holes through it.
   - **Purpose:** Simulates an AI Agent responding to a prompt like: *"Import that bracket, select the top base face, and drill four 5mm mounting holes into it."*

---

## Workflow 2: The "Live Web" Download & Edit
To truly prove the concept, we ran a script that actively queried GitHub, downloaded a real open-source CAD file, and edited it.

1. **`fetch_repo.py`**
   - **What it does:** Queries the GitHub REST API for the open-source **FreeCAD** repository, locates their internal test files directory, and downloads a real engineering component named `Schenkel.stp` (saving it as `downloaded_model.step`).
   - **Purpose:** Proves that an AI agent can autonomously source real CAD files from the internet.

2. **`cut_in_half.py`**
   - **What it does:** Natively imports the `downloaded_model.step` file, mathematically calculates the exact center of its bounding box, and uses a Boolean Split operation to slice the entire model perfectly in half along the Y-axis.
   - **Purpose:** Simulates an AI Agent responding to a prompt like: *"Take that downloaded STEP file and cut it exactly in half so I can 3D print it in two pieces."*

---

## Visualizing the Results
All of the `.step` files generated by these scripts were subsequently sent directly to Autodesk Fusion 360 via the local Model Context Protocol (MCP) server, which imported them and took high-resolution screenshots of the before-and-after results. You can view these images directly in the main project documentation.
